■ 1. システム概要
================================================================================

  アプリ名: シフト自動作成アプリ
  用途:     介護施設のシフト表を自動生成・管理する
  対象:     1F・2F のフロア別シフト管理
  動作環境: ブラウザ（Chrome/Edge推奨）— オフライン完全動作
  配布方法: dist/index.html 単体ファイル（ダブルクリックで起動）

■ 4. データモデル（types/index.ts）
================================================================================

4.1 型一覧
----------

  Floor        = '1F' | '2F'
  RoleType     = '正社員' | 'パート' | '派遣'
  RequestType  = '希望休' | '有給' | '出勤希望'
  PairType     = 'ng' | 'preferred'

4.2 Staff（スタッフ）
---------------------
  id                  : string      — 一意ID
  name                : string      — 氏名
  floor               : Floor       — 所属フロア
  role                : RoleType    — 雇用形態
  canLead             : boolean     — リーダー可否
  availableShiftTypes : string[]    — 対応可能シフトID配列
  monthlyWorkDays?    : number      — 月あたり出勤日数（パート等）
  unavailableDow      : number[]    — 出勤不可曜日（0=日, 6=土）
  memo                : string      — メモ

4.3 ShiftType（勤務種別）
-------------------------
  id           : string    — 固定ID（early/day/late/night/ake/training）
  name         : string    — 名称（早番/日勤/遅番/夜勤/明け/研修）
  shortName    : string    — 略称（L/B/U/〇/×/研）
  startTime    : string    — 開始時刻
  endTime      : string    — 終了時刻
  color        : string    — テキスト色
  bgColor      : string    — 背景色
  isDayShift   : boolean   — 日勤帯フラグ（早/日/遅 = true）
  isNightShift : boolean   — 夜勤フラグ（night = true）
  isAke        : boolean   — 明けフラグ（ake = true）
  order        : number    — 表示順

  デフォルト値:
    早番  id=early    08:00-16:30  青系    isDayShift=true
    日勤  id=day      08:30-17:00  緑系    isDayShift=true
    遅番  id=late     10:30-19:00  橙系    isDayShift=true
    夜勤  id=night    16:30-09:00  紫系    isNightShift=true
    明け  id=ake      -            灰系    isAke=true
    研修  id=training 09:00-17:00  水色    （手入力専用）

4.4 ShiftAssignment（シフト割当）
---------------------------------
  staffId     : string   — スタッフID
  date        : string   — 日付（YYYY-MM-DD）
  shiftTypeId : string   — シフト種別ID or 'off'
  isLeader    : boolean  — リーダーフラグ
  isManual?   : boolean  — 手入力フラグ（作り直し時の保護用）

4.5 FloorConfig（フロア設定）
-----------------------------
  floor              : Floor
  shiftRequirements  : Record<string, number>  — 各シフトの1日最低人数
  leaderCountPerDay  : number                  — 日勤リーダー人数/日
  maxConsecutiveDays : number                  — 最大連勤日数
  monthlyOffDays     : number                  — 月の公休日数

4.6 PairSetting（ペア相性）
---------------------------
  id       : string
  staffId1 : string
  staffId2 : string
  type     : PairType   — 'ng' | 'preferred'
  memo     : string

4.7 StaffDayComment（日別コメント）
-----------------------------------
  staffId : string
  date    : string
  comment : string

4.8 AppState（アプリ全体状態）
------------------------------
  staffList     : Staff[]
  shiftTypes    : ShiftType[]
  floorConfigs  : FloorConfig[]
  pairSettings  : PairSetting[]
  dayRequests   : DayRequest[]            ← 現在未使用（希望休ページ廃止のため）
  assignments   : ShiftAssignment[]
  staffComments : StaffDayComment[]
  currentFloor  : Floor
  currentYear   : number
  currentMonth  : number


■ 5. 状態管理（AppContext.tsx）
================================================================================

  方式: React Context + useReducer

  アクション一覧:
    SET_FLOOR          — フロア切替
    SET_MONTH          — 表示月切替
    SET_STAFF_LIST     — スタッフ一覧更新
    SET_SHIFT_TYPES    — シフト種別更新
    SET_FLOOR_CONFIGS  — フロア設定更新
    SET_PAIR_SETTINGS  — ペア設定更新
    SET_DAY_REQUESTS   — 希望休更新（未使用）
    SET_ASSIGNMENTS    — シフト割当更新
    SET_STAFF_COMMENTS — コメント更新
    DELETE_SHIFT_TYPE  — シフト種別削除（カスケード）
    RESTORE_ALL        — データ全復元（JSON読み込み時）

  永続化:
    各state変更時にuseEffectでLocalStorageに自動保存
    起動時にLocalStorageから自動読み込み

  マイグレーション:
    migrateShiftTypes() — デフォルトに存在するが保存データに無いシフト種別を自動追加


■ 6. ページ構成
================================================================================

6.1 スタッフ管理（StaffPage.tsx）
---------------------------------
  機能:
    - スタッフの追加・編集・削除
    - 名前、フロア、役職、対応可能シフト、リーダー可否、月出勤日数を設定
    - 並び替え（▲▼ボタン）
    - 削除時に関連データ（ペア設定、割当、コメント）をカスケード削除
    - 入力バリデーション（名前必須、シフト1つ以上、月出勤日数は正数）

6.2 シフト設定（SettingsPage.tsx）
----------------------------------
  機能:
    - フロアごとに設定（1F/2Fタブで切替）
    - 勤務種別一覧（読み取り専用表示）
    - 各シフトの1日最低人数
    - リーダー人数/日
    - 連勤上限日数
    - 月の公休日数
    - 明け翌日ルール説明
    - 入力値のclamp（範囲制限）

6.3 相性設定（PairPage.tsx）
----------------------------
  機能:
    - 夜勤ペアのNG / 推奨設定
    - 夜勤可能スタッフのみ選択可
    - 重複チェック
    - 追加・削除

6.4 シフト表（ShiftTablePage.tsx）
----------------------------------
  機能:
    - カレンダー形式のシフト表表示
    - セルクリックでサイクル入力: 空→休→日→夜→遅→早→研→空
    - 夜勤入力時に翌日の「明け」を自動挿入（翌日取消時に連動削除）
    - 手入力セルには isManual: true + 青ドット表示
    - 「空欄を自動で埋める」ボタン: 既存セルを保持し空欄のみ自動生成
    - 「作り直す」ボタン: 手入力(isManual)を保持し自動生成分だけリセット→再生成
    - 「クリア」ボタン: 全セル削除
    - 各スタッフの各日にコメント入力欄
    - 統計行: 出勤日数、夜勤回数、公休日数、リーダー回数
    - 最下行: 必要人数の充足状況（色分け表示）
    - Excel出力ボタン / PDF出力ボタン
    - 凡例表示


■ 7. スケジューラー（scheduler.ts）
================================================================================

  アルゴリズム: ルールベース制約充足 + 貪欲法（日ごと処理）

7.1 処理フロー
--------------
  入力: スタッフ一覧, シフト種別, フロア設定, ペア設定, 前月割当, 事前入力

  1日ずつ以下のフェーズを実行:

    Phase 0  — 夜勤→明け強制（最優先ハードルール）
              前日が夜勤のスタッフは無条件で明けを割当。事前入力より優先。

    Phase 0.5— 事前入力の適用
              手入力されたセルを反映。ただし明けの翌日制約に違反する場合は棄却。

    Phase 1  — 強制休み
              希望休・有給・出勤不可曜日による休み割当。

    Phase 2  — 夜勤配置
              必要人数に満たない場合、候補者から選出。
              ペア相性（NG除外・推奨優先）を考慮。
              夜勤回数が少ない人を優先。
              明け翌日の人も夜勤候補になる。
              明け翌日で夜勤にも入らなかった人は休みを割当。

    Phase 3  — 日勤帯シフト配置
              不足しているシフト種別を最大不足順に1人ずつ埋める。
              候補者ソート: 公休ペース遅れ小 > 公休達成済み > 勤務日数少ない順。

    Phase 4  — リーダー配置
              日勤に入っているリーダー可スタッフから選出。
              リーダー回数が少ない人を優先。

    Phase 5  — 残り未割当の処理
              「残り公休必要数 / 残り日数」vs「残り勤務必要数 / 残り日数」で
              比率が高い方を優先（公休が遅れていれば休み、勤務が遅れていれば出勤）。
              連勤上限・月間出勤上限も考慮。

7.2 夜勤ペア選出（selectNightWorkers）
--------------------------------------
  - 夜勤回数少ない順でソート
  - NG ペアを含む組み合わせは除外（-10000点）
  - 推奨ペアに +10点ボーナス
  - 最大12人の候補プールからk人の全組み合わせを評価

7.3 制約一覧
-------------
  ハード制約（必ず守る）:
    - 夜勤翌日は明け（事前入力より優先）
    - 明け翌日は夜勤 or 休みのみ
    - 連勤上限
    - 月間出勤上限（個人設定がある場合）
    - 対応不可シフトには配置しない
    - NGペアは同じ夜勤に入れない

  ソフト制約（可能な範囲で守る）:
    - 月の公休日数を全スタッフが達成
    - 夜勤回数の均等化
    - 推奨ペアの優先
    - 各シフトの必要人数充足
    - 勤務日数の均等化


■ 8. 出力機能
================================================================================

8.1 Excel出力（excelExport.ts）
-------------------------------
  ライブラリ: xlsx (SheetJS)
  出力内容:
    - スタッフ名, 日付ヘッダ, シフト略称, 統計列（出勤/夜勤/公休）
    - コメント行（コメントがあるスタッフの下に日別コメントを表示）
    - 列幅調整済み

8.2 PDF出力（pdfExport.ts）
---------------------------
  方式: HTMLテーブルを生成 → 別ウィンドウで window.print()
  出力内容: Excel出力と同等（コメント行含む）
  印刷用CSS適用（A4横向き）


■ 9. データ入出力（dataIO.ts）
================================================================================

  エクスポート:
    - AppState の主要データを JSON ファイルとしてダウンロード
    - ファイル名: シフトデータ_YYYYMMDD.json
    - _format / _version によるフォーマット識別

  インポート:
    - JSONファイルを読み込んでデータ復元
    - フォーマットチェック + 必須キーの存在確認
    - RESTORE_ALL アクションで全state上書き


■ 10. UI構成（Layout.tsx）
================================================================================

  構成:
    - 左サイドバー（ナビゲーション + データ書出/読込ボタン）
    - メインエリア（ヘッダー + コンテンツ）
    - ヘッダーに月切替（← →）、フロアタブ（1F/2F）

  ナビゲーション:
    staff    → スタッフ管理
    settings → シフト設定
    pairs    → 相性設定
    shift    → シフト表

  サイドバー下部:
    - 「データ保存（書き出し）」ボタン → exportAppData
    - 「データ読み込み」ボタン → importAppData → RESTORE_ALL
