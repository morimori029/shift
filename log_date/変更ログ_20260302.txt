シフト管理アプリ 修正・変更ログ
作成日: 2026-03-02
===============================================================

■ 前回セッション（2026-02-24 〜 25）
---------------------------------------------------------------

1. バグ修正：XSS 脆弱性の解消
   - dangerouslySetInnerHTML を使っていた箇所を安全な React ノード描画に変更
   - 対象: ShiftTablePage.tsx, dataIO.ts, Layout.tsx

2. バグ修正：夜勤警告判定の修正
   - 夜勤不足の警告条件が「< 2」固定になっていたのを、
     設定値（nightReq）と比較するように修正
   - 対象: ShiftTablePage.tsx

3. バグ修正：インポートデータの型検証強化
   - インポート時に不正なデータでアプリがクラッシュしないよう
     各フィールドの形状チェックを追加
   - 対象: dataIO.ts

4. 新機能：祝日設定
   - 全フロア共通の祝日リストを設定できるように
   - AppState に holidays 配列を追加
   - SET_HOLIDAYS アクションを追加
   - シフト表で祝日セルを赤背景・「祝」マークで強調表示
   - スケジューラーが祝日を考慮して必要人数を計算
   - 対象: types/index.ts, AppContext.tsx, scheduler.ts,
           ShiftTablePage.tsx, SettingsPage.tsx, dataIO.ts, defaults.ts

5. 新機能：シフト必要人数の有効/無効チェックボックス
   - FloorConfig に shiftRequirementsEnabled フィールドを追加
   - 各シフト種別ごとにON/OFFで自動生成での必要人数適用を切り替え可能に
   - 既存データはデフォルト全てON でマイグレーション
   - 対象: types/index.ts, defaults.ts, AppContext.tsx,
           scheduler.ts, SettingsPage.tsx


■ 本日（2026-03-02）
---------------------------------------------------------------

1. バグ修正：ビルドエラー解消
   - scheduler.test.ts の BASE_CONFIG に shiftRequirementsEnabled を追加

2. 新機能：祝日の必要人数設定（フロアごと）
   - FloorConfig に holidayShiftRequirements・useHolidayRequirements を追加
   - チェック OFF（デフォルト）: 祝日はその日の実際の曜日の必要人数をそのまま使用
   - チェック ON: シフト種別ごとに祝日専用の必要人数を入力可能
   - 対象: types/index.ts, scheduler.ts, SettingsPage.tsx

3. UI 削除：シフト必要人数テーブルの「有効」チェックボックスを削除
   - ユーザー要望により除去
   - データ・スケジューラー側のロジックはそのまま残す
   - 対象: SettingsPage.tsx

4. UI 改善：業務選択をプルダウン（select）に変更
   - 早番・日勤セルの業務行をクリックで循環する方式から
     ドロップダウンリストで直接選択する方式に変更
   - 対象: ShiftTablePage.tsx

5. 表記変更：「1F」→「応援」
   - DutyType: 'onef' の表示ラベルを「1F」から「応援」に変更
   - 応援の人が必要人数カウントから除外される動作は継続
   - 対象: types/index.ts（DUTY_LABELS）

6. 仕様変更：クリアボタンの挙動変更
   - 「クリア」ボタンが全削除していたのを、
     手入力（isManual=true）のシフトを残し自動生成分のみ削除するように変更
   - 対象: ShiftTablePage.tsx

7. 新規作成：HTMLマニュアル
   - dist/manual.html を作成（操作手順書）
   - 全機能を網羅した8章構成

8. バグ修正：インポートの後方互換対応
   - 旧バージョンのJSONには holidays キーが含まれていないため
     読み込めなかった問題を修正
   - holidays を「省略可能キー」として扱うことで旧JSONも読み込み可能に
   - 対象: dataIO.ts

9. 仕様変更：祝日チェックOFF時の動作変更
   - チェックOFF時に祝日を「日曜日（dow=0）」扱いしていた動作を廃止
   - チェックOFF = 祝日でもその日の実際の曜日の必要人数をそのままにする
   - チェックON = 祝日専用の入力値を使う
   - 対象: scheduler.ts

10. 更新：仕様書
    - 仕様書.md をバージョン 1.4 → 1.6 に更新（最終更新: 2026-03-02）
    - 上記の変更内容をすべて反映
